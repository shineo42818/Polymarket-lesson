## Why Did the Bot Stop? (Hypotheses)

### #1 — Daily Loss Kill Switch (Most Likely)
- `engine.py:419-421`: kills engine when `daily_pnl <= -$15.00`
- Root cause: **over-aggressive accounting bug** in `order_manager.py:476`
  - When a PARTIAL trade expires and the market hasn't resolved yet:
    `trade.settled_pnl = -filled_usdc`  (worst-case write-off)
  - One partial fill leg ≈ $5. Three of these in a day = -$15 → kill switch fires
  - The market may resolve in your favor later, but the bot already counted it as a total loss
- Paper sim has 50% maker fill probability → half of all trades become PARTIAL → frequent worst-case write-offs

### #2 — systemd Restart Limit
- If uvicorn crashed repeatedly, systemd stops retrying
- Check: `sudo systemctl status bot` → look for `"start-limit-hit"`

### #3 — OOM Kill (Least Likely)
- e2-small has 2 GB RAM; workload is light, but possible memory leak
- Check: `sudo dmesg | grep -i "killed process"`

```bash
for table in ['trades', 'portfolio_snapshots']:
    rows = conn.execute(f'SELECT * FROM {table} ORDER BY id').fetchall()
    if not rows:
        print(f'{table}: empty'); continue
    with open(f'data/{table}.csv', 'w', newline='') as f:
        w = csv.DictWriter(f, fieldnames=rows[0].keys())
        w.writeheader()
        w.writerows([dict(r) for r in rows])
    print(f'{table}: {len(rows)} rows -> data/{table}.csv')
conn.close()
"
```
